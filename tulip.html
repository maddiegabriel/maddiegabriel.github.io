About Tulip



Tulip is a mid-sized software startup providing a SaaS-based mobile platform for retailers. The goal of Tulip's product is to empower retail stores to better connect their customer and store associate experiences. Tulip's app is widely used on ipads by store associates, mainly at luxury retailers, to assist customers with their in-store shopping experience.

Tulip has two offices - one in downtown Toronto and one in Kitchener. I worked from the Toronto office for my co-op term. There are many different software teams at Tulip, each working on different portions of the Tulip platform. I worked with the Checkout Nova product team during my work term!

I chose to work at Tulip because I knew I would learn tons of new technical skills by working at a company centred around tech. I also wanted to try working at a smaller company than I have in the past!

My Role

Software Developer


Overview

For my co-op I was a dev on Tulip's Checkout Nova product team, made up of four devs, two QA Analysts, a Project Manager and a Scrum Master. My team worked on the Checkout app in the Tulip platform, which allows customers to pay for their purchases in-store on a store associate's ipad. This involves actions like building a cart, calculating regional taxes, processing payments, storing order details, and handling returns/refunds. I worked on a few main Checkout projects during my term, which I've explained below. Other than these projects, I did bug-fixes for our app, demoed new features at Sprint Reviews, pair-programmed with other developers, and worked with different teams when they needed an extra hand. I also spent time doing a lot of self-learning, since my work involved many tools and technologies I had never used before!


Main projects I worked on this term:


1) Admin Console Return Window Feature
  • Admin Console is our web app which allows store managers to configure their store's Tulip platform. One thing managers can customize through Admin Console is their store's ability to process returns & refunds.
  • For this project, I was responsible for building a new feature in Admin Console which allows store managers to configure and edit their return & refund preferences.
  • First, I worked with our UX designers using Figma to understand and improve the feature design.
  • I learned React on the job top build the frontend of this feature from scratch. This involved adding new re-usable components to the Tulip React library and architecting my solution to be extensible.
  • Building on our existing internal API, I wrote the backend code for this feature in PHP. This involved sending, receiving, validating and updating data between our database and the Admin Console frontend.
  • In Sprint Reviews, I was able to demo this feature many times while it was in development. This was great practice in accepting feedback and iterating on my design.
  • I really enjoyed working on this project from start to finish, and I'm happy it will be used in production when the next app version launches!
2) Tulip's internal Taxman API
  • One area of the Checkout app which ended up being more complicated than I anticipated was the calculation of taxes on an order. Taxes are tricky because they're regional and are subject to change based on government legislation.
  • During my co-op, I assisted in developing an internal Tulip API called Taxman. This API acts as an interface between our app and an external tax service to calculate accurate tax amounts for orders based on store location.
  • The work I did on Taxman was mostly in PHP. I also made some structural database changes using MySQL. I learned a lot about database migrations and how to break down a complicated problem.
  • This project was fun to work on since I got to do a lot of pair-programming with another dev on my team!
3) Improve the local development environment at Tulip
  • During my co-op, I noticed a lack of strong internal documentation for developers, especially around the setup of our local environments. This made it difficult to get onboarded at the start, since there are so many different tools and technologies involed and a lot can go wrong.
  • To fix this problem, I created some new onboarding documentation and FAQ pages to help ease the process for current and future developers at Tulip.
  • During this process, I worked with incoming developers and sales engineers to help them set up their local development environments. This helped me recognize pain points in the setup and improve the documentation in difficult areas.

Semester Goals

Tech Literacy

My main goal at Tulip was to learn a bunch of new technical skills. I hoped to gain experience doing both front & backend development. I achieved this and learned many technologies along the way, namely: PHP, React, Docker, MySQL, and JavaScript. I worked hard to learn new skills and overcame plenty of technical challenges, especially during my onboarding learning curve. I tried to tackle my work with curiosity, learning independently and asking for help when needed. I learned so much from the experienced devs on my team! By March, I was comfortanble with my dev environment and able to fix issues. I used feedback from code reviews to help me improve. As an example of my technical learning, I learned React from scratch to successfully complete a production feature. I also did a Hack Day project using React to solidify my learning.

Teamwork

During this term, I reached my teamwork goal by gaining experience working on an agile dev team. I think I became a helpful member of the Checkout team at Tulip! To reach this goal, I paid attention to team practices so I could contribute better during my work term. I tried to work independently where I could, but asked my team for help when I needed it rather than getting stuck on one task forever. I always tried to bring an enthusiastic attitude to the team and help out wherever I could! The quarantine was definitely an interesting interruption to our team dynamic, but I think we overcame the challenge of transitioning to remote work really well. I met some really amazing developers during my time at Tulip who taught me so much. In the future, I could improve by making more professional connections outside of my immediate team!

Information Literacy

This term, I feel I really worked at achieving my Information Literacy goal. I was thrown onto many projects where I had never worked with the tools or technologies before. This means I was faced with a lot of heavy technical documentation to parse through. By the end of my co-op I was able to ramp up way more quickly on new technologies and troubleshoot issues much faster. This is because, through practice, I became more comfortable at parsing information in technical documentation to find the answers I need. I really tried to understand the tools I was using this semester, instead of taking shortcuts. I also got the chance to improve Tulip's internal dev documentation, which allowed me to practice writing technical documentation. I feel Information Literacy is a super valuable skill, especially for my future career as a developer!


Conclusion

My work term at Tulip was a super valuable addition to my co-op experiences at U of G! Through my role as a Software Developer, I was able to learn so many new technical skills while working with and learning from some awesome experienced developers. Through mentorship, I gained confidence in my ability to overcome technical challenges and learn new skills quickly as part of a fast-paced development team. In my time at Tulip I gained valuable skills and experience which I can apply during my last year at Guelph and beyond!
